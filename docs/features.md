# Auto Backup and Patch Tool - 功能文档

## 1. 软件概述

Auto Backup and Patch Tool 是一款基于 Python 和 Qt GUI 的桌面应用程序，用于安全地部署文件补丁（将一个目录的内容复制到另一个目录），并提供自动备份和回滚能力。支持**本地**和**远程（SSH）**目录操作。

### 1.1 技术栈
- **语言**: Python 3
- **GUI 框架**: PySide2（优先）/ PySide6（降级兼容）
- **远程传输**: paramiko（SSH/SFTP）
- **配置文件**: JSON 格式

### 1.2 项目结构

```
├── conf/               # 配置文件目录
│   ├── config.json     # 默认配置文件
│   └── config_test.json # 测试配置文件
├── gui/                # Qt 图形用户界面
│   ├── main_window.py  # 主窗口与业务逻辑
│   └── icons/          # SVG 图标资源
├── lib/                # 核心功能库
│   ├── backup_lib.py   # 备份/补丁/回滚核心引擎
│   └── password_manager.py # SSH 密码管理器
├── logs/               # 应用日志（按天存储）
├── tests/              # pytest 测试套件
│   └── test_backup_lib.py
├── start.sh            # 启动脚本
└── requirements.txt    # Python 依赖说明
```

## 2. 主要功能模块

### 2.1 核心引擎 (`lib/backup_lib.py`)

提供所有底层文件操作能力，支持四种组合模式：本地→本地、本地→远程、远程→本地、远程→远程。

#### 2.1.1 补丁 (Patch)
- **功能**: 将 `output_dir` 的内容复制到 `target_dir`，覆盖现有文件
- **API**: `patch(output_dir, target_dir, ...)`
- **特点**:
  - 自动忽略隐藏文件（如 `.DS_Store`, `.git`）
  - 支持目标目录不存在时自动创建
  - 远程操作通过 SFTP 上传/下载

#### 2.1.2 回滚 (Rollback)
- **功能**: 将备份目录恢复到 `target_dir`
- **API**: `rollback(backup_dir, target_dir, ...)`
- **特点**:
  - 支持从任意历史备份恢复
  - 与补丁操作共享底层复制逻辑

#### 2.1.3 备份 (Backup)
- **功能**: 创建带时间戳的完整备份
- **API**: `backup(target_dir, backup_dir, ...)`
- **命名格式**: `{basename}_YYYYMMDD_HHMMSS`
- **特点**:
  - 本地与远程互传均支持
  - 空备份目录自动清理

#### 2.1.4 兼容性检查 (`check_patch_compatibility`)
- **功能**: 补丁/回滚前的安全预检，比较源目录和目标目录的顶层结构
- **返回状态**:
  - `"match"` — 安全，所有顶层项重叠
  - `"partial"` — 部分差异，需用户确认
  - `"none"` — 完全不同，禁止操作
  - `"empty_target"` — 目标为空/不存在
  - `"remote"` — 目标为远程目录（基础检查）

#### 2.1.5 重叠文件检测与选择性备份
- **`find_overlapping_paths`**: 检测将被覆盖的文件/目录
- **`backup_overlapping_files`**: 仅备份会被覆盖的部分（本地目标），远程目标则执行完整备份
- **优化**: 仅保留最底层的重叠项，避免不必要的冗余复制

#### 2.1.6 远程目录支持
- **路径格式**: `user@host:/path`
- **功能**:
  - 远程目录浏览与列表
  - SFTP 上传/下载
  - 远程命令执行
  - 远程到远程的传输（通过本地临时目录中转）

---

### 2.2 密码管理器 (`lib/password_manager.py`)

- **密码缓存**: 按 `user@host` 缓存 SSH 密码，持久化到 JSON 配置文件的 `ssh_passwords` 字段
- **交互式提示**: 未缓存时弹出 Qt 密码输入对话框
- **密码验证**: 通过 paramiko 实际连接验证密码正确性
- **重试机制**: 验证失败自动清除缓存并重新提示

---

### 2.3 图形用户界面 (`gui/main_window.py`)

#### 2.3.1 主窗口布局
- **三个目录输入框**（均支持本地路径、`~` 符号自动解析为用户主目录，以及 `user@ip:/path` 远程路径）:
  - Backup Dir — 备份存储目录
  - Output Dir — 补丁源目录
  - Target Dir — 补丁目标目录
- **浏览按钮**: 本地路径调用系统文件对话框；远程路径调用内置远程目录浏览器
- **操作按钮**: Patch（打补丁）、Rollback（回滚）
- **日志窗口**: 实时显示操作日志，带时间戳

#### 2.3.2 远程目录浏览器 (`RemoteDirDialog`)
- 通过 SSH 连接浏览远程文件系统
- 支持双击进入子目录、返回上级、刷新列表
- 文件与目录通过图标区分
- 异步加载，不阻塞界面

#### 2.3.3 操作确认对话框
- **Markdown/HTML 渲染**: 以表格形式展示结构差异和将被覆盖的文件列表
- **部分匹配提示**: 显示两侧差异项（最多 10 项），用户确认后才继续
- **自定义 SVG 图标**: 替换系统默认消息框图标

#### 2.3.4 配置管理
- **保存参数**: 将当前输入的目录路径和密码缓存保存为 JSON 文件
- **加载参数**: 从 JSON 文件恢复配置
- **默认配置**: 启动时自动加载 `conf/config.json`

#### 2.3.5 后台线程
- 所有网络与文件操作在 `QThread` 中执行
- 预检线程 (`_PreCheckThread`)：执行兼容性检查和重叠检测
- 工作线程 (`_WorkerThread`)：执行实际的补丁/回滚/备份操作
- UI 保持响应，支持忙状态光标和按钮禁用

---

### 2.4 日志系统
- **实时显示**: 主界面日志窗口显示带时间戳的操作记录
- **文件持久化**: 按天存储到 `logs/YYYY-MM-DD.log`
- **内容**: 包括操作步骤、错误信息、用户决策等

## 3. 核心操作流程

### 3.1 打补丁 (Patch) 流程

1. **输入验证** — 检查 Output 和 Target 是否为空；验证 Output 目录存在
2. **密码获取** — 若涉及远程路径，通过密码管理器获取并验证 SSH 密码
3. **兼容性预检** — 后台线程比较源目录和目标目录的顶层结构
   - `"none"` → 弹出禁止对话框，终止操作
   - `"partial"` → 弹出 Markdown 对比表格，需用户确认
   - `"match"` / `"empty_target"` → 直接通过
4. **重叠文件展示** — 列出将被覆盖的文件/目录（最多显示 20 项），用户二次确认
5. **备份重叠文件** — 将目标端将被覆盖的部分备份到 Backup Dir
6. **执行补丁** — 将 Output 复制到 Target
7. **完成提示** — 弹出成功对话框

### 3.2 回滚 (Rollback) 流程

1. **输入验证** — 检查 Backup 和 Target 是否为空
2. **密码获取** — 同补丁流程
3. **列出备份** — 扫描 Backup Dir 中带时间戳的备份目录，按时间倒序排列
4. **选择备份** — 用户从下拉列表中选择要恢复的备份版本
5. **兼容性预检** — 同补丁流程（比较备份与目标目录）
6. **重叠文件展示与确认** — 同补丁流程
7. **执行回滚** — 将选定备份复制到 Target
8. **完成提示**
